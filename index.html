7-Week React Course

This course will take you through the fundamentals of React, focusing on core concepts and practical application. We'll build upon each week, culminating in a solid understanding of how to create and manage React applications.

Week 1: Introduction to React and Components

Topics:

  • Functional Components
  • JSX
  • Props vs State
  • Conditional Rendering
  • Composition
  • Component Basics

Materials/Tutorials:

  • React Official Documentation - Components and Props
  • React Official Documentation - JSX
  • W3Schools React Tutorial
  • FreeCodeCamp React Curriculum

Activities:

  • Create your first functional component.
  • Practice passing data using props.
  • Build a simple UI with conditional rendering.

Assignments:

Assignment 1: Create a component that displays a greeting message based on the time of day. Use props to customize the greeting.

Projects:

Project 1: Build a simple to-do list application. Users should be able to add, remove, and mark tasks as complete.

Week 2: Working with State and Effects

Topics:

  • useState
  • useEffect
  • Basic Hooks

Materials/Tutorials:

  • React Official Documentation - Using the State Hook
  • React Official Documentation - Using the Effect Hook
  • Kent C. Dodds' Blog - `useEffect` Deep Dive
  • "Dave Gray" on YouTube - React Hooks Tutorial (Search on YouTube)

Activities:

  • Build an interactive component using useState.
  • Fetch data from an API using useEffect.
  • Implement a counter using hooks.

Assignments:

Assignment 2: Create a component that fetches data from a public API (e.g., a random quote API) and displays it. Use `useEffect` to handle the data fetching.

Projects:

Project 2: Enhance your to-do list application to save the tasks to local storage so they persist across browser sessions.

Week 3: Lists, Keys, and Forms

Topics:

  • Lists and Keys
  • Forms

Materials/Tutorials:

  • React Official Documentation - Lists and Keys
  • React Official Documentation - Forms
  • "Codecademy" React Forms Tutorial (Search on YouTube)

Activities:

  • Render a list of items from an array.
  • Build a form to capture user data.
  • Implement controlled components.

Assignments:

Assignment 3: Create a form that allows users to add new items to a list. Display the updated list whenever a new item is added. Use controlled components for the form inputs.

Projects:

Project 3: Build a simple e-commerce product listing page. Display a list of products (you can use placeholder data). Each product should have an image, name, price, and a "Add to Cart" button. (Don't worry about actual cart functionality for this project, just the UI).

Week 4: Context and Advanced Hooks

Topics:

  • useContext
  • useRef
  • useCallback
  • Common Hooks (Review)

Materials/Tutorials:

  • React Official Documentation - Context
  • React Official Documentation - useRef
  • React Official Documentation - useCallback
  • "The Net Ninja" on YouTube - React Context API Tutorial (Search on YouTube)

Activities:

  • Implement a theme switcher using Context API.
  • Build a component that focuses an input on mount using useRef.
  • Optimize a component using useCallback.

Assignments:

Assignment 4: Create a component that manages a user's theme preference (light/dark mode) using Context API. The theme should be applied to the entire application.

Projects:

Project 4: Build a simple form with several input fields. Use `useRef` to automatically focus the first input field when the component mounts. Use `useCallback` to optimize any functions that are passed down as props to child components.

Week 5: Routing with React Router

Topics:

  • React Router

Materials/Tutorials:

  • React Router Official Documentation
  • "PedroTechnologies" on YouTube - React Router Tutorial (Search on YouTube)

Activities:

  • Build a multi-page application with React Router.
  • Implement navigation and dynamic routes.

Assignments:

Assignment 5: Convert your to-do list application (from Week 1 or 2) into a multi-page application using React Router. Create separate pages for adding tasks, viewing tasks, and a "home" page.

Projects:

Project 5: Create a simple blog application with multiple pages (e.g., home, about, contact, and individual blog post pages). Use React Router to handle navigation. You can use placeholder data for the blog posts.

Week 6: State Management with Context (and potentially Zustand/Jotai)

Topics:

  • Context (Deep Dive)
  • Zustand/Jotai (Optional)

Materials/Tutorials:

  • "UseContext vs Redux" articles (Search online)
  • Zustand Documentation (If covering Zustand)
  • Jotai Documentation (If covering Jotai)

Activities:

  • Manage application-wide state using Context.
  • Implement a simple store with Zustand or Jotai (optional).

Assignments:

Assignment 6: Refactor your e-commerce product listing application (from Week 3) to manage the cart state using Context API. Implement the "Add to Cart" functionality to update the cart count in the header.

Projects:

Project 6: Build a more complex application (e.g., a simple social media feed, a recipe app, or a small dashboard) that requires managing global state. Use Context API (or Zustand/Jotai if you covered it) for state management.

Week 7: Testing and Building

Topics:

  • Vitest / Jest
  • Building

Materials/Tutorials:

  • Vitest Documentation (If using Vitest)
  • Jest Documentation (If using Jest)
  • React Official Documentation - Production Build
  • "Testing JavaScript" by Kent C. Dodds

Activities:

  • Write unit tests for your components.
  • Learn how to create a production build.

Assignments:

Assignment 7: Write unit tests for the key components of your final project. Focus on testing the functionality and interactions of the components.

Projects:

Project 7: Prepare your final project for deployment by creating a production build. If possible, deploy your project to a hosting platform (e.g., Netlify, Vercel, or GitHub Pages).

Final Project Suggestions

  • Personal Portfolio Website: A single-page website showcasing the student's skills, projects, and contact information. Focus on clean design, responsive layout, and using components effectively.
  • Simple E-commerce Storefront (Simplified): Display a catalog of products (using mock data or a simple API integration). Focus on dynamic lists, conditional rendering, and potentially a basic cart feature using Context API.
  • Interactive Quiz App: Create a quiz with multiple-choice questions, scoring, and feedback. Focus on state management, event handling, and conditional rendering.
  • Basic Task Management App (Enhanced): Go beyond the simple to-do list. Add features like due dates, categories, prioritization, and local storage persistence.
  • Blog Application (with Content Management): A blog with multiple pages (home, about, contact, blog posts). Implement routing, dynamic content loading (mock data or a simple CMS integration), and potentially a basic commenting system.
  • Recipe App: Fetch recipes from an API (e.g., The Meal DB) and display them with filtering and search capabilities. Focus on API integration, data handling, and potentially using a state management library like Zustand or Jotai.
  • Simple Social Media Feed (Simplified): A feed displaying posts (mock data or simple API). Implement features like posting, liking, and potentially following/followers (simplified). Focus on state management and data flow.
  • Real-time Chat Application (Basic): Use a simple backend service (or a service like Firebase) to create a chat application. Focus on handling real-time data, user authentication (simplified), and UI updates.
  • Full-Stack E-commerce Application (Miniaturized): Integrate with a backend (you can provide a simple one or guide students to create one) for product data, user authentication, and order processing (simplified). Focus on data fetching, state management, routing, and potentially payment gateway integration (mock or simplified).
  • Project Management Dashboard: Create a dashboard for managing tasks, projects, and team members (mock data or simple backend). Focus on complex state management, data visualization (using libraries like Chart.js), and user interface design.
  • Custom API Integration Project: Choose a public API (e.g., weather, news, movies) and build an application around it. Focus on API interaction, data processing, and presenting information effectively.
